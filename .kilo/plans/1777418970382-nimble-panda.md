# Profit Calculation Logic Update Plan

## Problem Analysis
The current application calculates profit as:
- Profit = Sales - Expenses

The user wants to update this to:
- Gross Profit = Sales - Purchases
- Net Profit = Gross Profit - Other Expenses - Tax

Where:
- Sales: Transactions with type='sale'
- Purchases: Transactions with type='expense' that have inventory_item_id (these affect inventory)
- Other Expenses: Transactions with type='expense' that do NOT have inventory_item_id
- Tax: Not currently tracked in the system (needs clarification)

## Current Implementation Review
### Routes (routes/index.js)
1. Home route ('/') - lines 194-245:
   - Calculates todaySales, todayExpenses, todayProfit
   - Calculates monthSales, monthExpenses, monthProfit
   - Uses getSumByTypeAndDate and getSumByTypeAndDateRange helpers

2. Reports route ('/reports') - lines 705-843:
   - Calculates daily and monthly summaries
   - Calculates totalSales, totalExpenses, totalProfit
   - Groups transactions by type to separate sales vs expenses

### Models (models/index.js)
- Transaction model has type field ('sale' or 'expense')
- Transaction model has inventory_item_id field (null when not inventory-related)
- applyInventoryTransactionImpact function shows:
  - type='sale' decreases inventory (delta = -quantity)
  - type='expense' increases inventory (delta = quantity) when inventory_item_id exists
  - This confirms that 'expense' with inventory_item_id = purchase transactions

## Solution Approach
1. Create new helper functions in models/index.js:
   - getSumOfSales(businessId, date/dateRange)
   - getSumOfPurchases(businessId, date/dateRange) - expenses with inventory_item_id
   - getSumOfOtherExpenses(businessId, date/dateRange) - expenses without inventory_item_id

2. Update routes/index.js to use these new functions:
   - Replace current sales/expenses calculations with:
     - sales = getSumOfSales(...)
     - purchases = getSumOfPurchases(...)
     - otherExpenses = getSumOfOtherExpenses(...)
     - grossProfit = sales - purchases
     - netProfit = grossProfit - otherExpenses (tax to be determined)

3. Update the reports route similarly to show:
   - Gross Profit = Sales - Purchases
   - Net Profit = Gross Profit - Other Expenses

   - Should tax be a separate transaction type?

2. Timing: When should these calculations update? 
   - The user said "so that it can update automatically by simply entering a transaction"
   - This suggests the calculations should happen in real-time as transactions are added/edited
   - Current implementation recalculates on each page load, which should be sufficient
